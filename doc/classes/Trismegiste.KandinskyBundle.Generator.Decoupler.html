  • \Decoupler

Decoupler is a service which generates 2 classes and 1 interface based on one specified concrete class.

The goal is to achieve loosely coupling by replacing this class with the generated interface => Liskov Substitution Principle

Better : if you explode this interface into multiple little interfaces, you can achieve the Interface Segregation Principle. This is the key to refactoring for achieving the Single Responsibility Principle

You can use the analytics tool to figure out how to explode the generated interface.
